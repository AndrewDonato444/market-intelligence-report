## Feature: Copyright notices on generated PDF reports

Every exported PDF intelligence brief must carry proper copyright attribution and a confidentiality disclaimer. This protects Modern Signal Advisory's intellectual property on all distributed outputs — whether shared digitally or printed — without cluttering the advisory content the report reader depends on.

The copyright line and confidentiality notice must feel like an institutional standard, not an afterthought. Think Bloomberg disclaimer bar — present, professional, ignorable until you need it.

### Scenario: Copyright line appears on every page footer
Given a report PDF is generated
When the PDF is rendered
Then every page (cover, table of contents, insights index, and all section pages) displays "© {current year} Modern Signal Advisory. All rights reserved." in the page footer area
And the copyright text uses `color-text-tertiary` (#94A3B8) at 7pt Inter
And it does not compete visually with the report title or page number already in the footer

### Scenario: Copyright line coexists with existing footer elements
Given the page footer already shows the report title (left) and page number (right)
When the copyright line is added
Then the copyright text appears as a second row below the existing footer rule
And the report title and page number remain in their current positions
And vertical spacing between the footer rule and copyright line is minimal (4pt)

### Scenario: Confidentiality disclaimer on cover page
Given a report PDF is generated
When the cover page is rendered
Then a confidentiality notice appears near the bottom of the cover page
And it reads: "Confidential. This intelligence brief is prepared exclusively for the use of the commissioning advisor and their designated clients. Redistribution without written consent of Modern Signal Advisory is prohibited."
And it uses 7pt Inter in `color-text-secondary` (#475569) on the dark cover background — rendered in `color-text-inverse` (#F8FAFC) with reduced opacity

### Scenario: Confidentiality disclaimer on final page
Given a report PDF has been fully rendered with all sections
When the last section page is rendered
Then a confidentiality block appears after the section content on the final page
And it includes the same confidentiality text as the cover page
And it includes: "Report generated by Modern Signal Advisory intelligence platform. Data sources include public records, MLS feeds, and third-party analytics. This report does not constitute legal, financial, or appraisal advice."
And it uses 7pt Inter in `color-text-tertiary`

### Scenario: Year in copyright updates automatically
Given the current year changes (e.g., from 2026 to 2027)
When a new PDF is generated
Then the copyright line reflects the current year
And no code change is required

### Scenario: Copyright does not appear in web report view
Given the user is viewing a report in the browser (not PDF)
When the report sections are rendered
Then no PDF-specific copyright line or confidentiality disclaimer appears in the web UI
And the existing web disclaimer component remains unchanged

### Scenario: Agent branding does not override copyright
Given an agent has custom brand colors configured
When their report PDF is generated
Then the copyright line still reads "Modern Signal Advisory" (not the agent's company name)
And the confidentiality disclaimer still references "Modern Signal Advisory"
And agent branding applies only to the agent attribution areas, not the platform copyright

## User Journey

1. Agent creates a market intelligence report in the app
2. Agent clicks "Download PDF" to export the brief
3. **PDF includes copyright + confidentiality notices on every page and cover**
4. Agent shares PDF with clients — IP is protected, platform is attributed
5. Client sees professional institutional attribution (builds trust, not distraction)

## Implementation Notes

### Files to modify
- `lib/pdf/styles.ts` — Add `copyrightText` and `confidentialityBlock` styles
- `lib/pdf/templates/section-page.tsx` — Add copyright line below existing `pageFooter`
- `lib/pdf/templates/table-of-contents.tsx` — Add copyright line below existing footer
- `lib/pdf/templates/insights-index.tsx` — Add copyright line below existing footer
- `lib/pdf/templates/cover-page.tsx` — Add confidentiality notice + copyright to cover
- `lib/pdf/document.tsx` — Pass `isLastSection` flag to final SectionPage for end-of-report disclaimer block

### Design token usage
- Copyright text: `color-text-tertiary` (#94A3B8), 7pt Inter
- Confidentiality block on final page: `color-border` (#E2E8F0) for border, `color-text-tertiary` for text
- Confidentiality on cover: `color-text-inverse` (#F8FAFC) at reduced opacity on dark background
- No new design tokens required — uses existing report palette

### What this does NOT include
- Visible watermark overlay across page content (would harm readability — anti-persona concern)
- DRM or copy-protection on the PDF file
- Agent-specific copyright (always "Modern Signal Advisory")

## Persona Revision Notes

**Report Reader lens**: The copyright and disclaimer must not add visual noise. The report reader skips anything that feels like filler — so the copyright line is tertiary-colored, small, and tucked into the footer where it's visible but not competing with intelligence content. The confidentiality notice on the cover adds institutional gravitas ("this is a real advisory document") without being a compliance wall of text.

## Learnings

### 2026-03-29
- **Pattern**: Shared copyright constants live in `lib/pdf/copyright.ts` — a PDF-only module. This ensures web UI never accidentally imports copyright text.
- **Pattern**: `getCopyrightLine()` is a function (not a const) so the year is always current at render time, not at module load time.
- **Decision**: `isLastSection` prop is computed in `document.tsx` via array index comparison (`index === filteredSections.length - 1`) and passed to `SectionPage`. This keeps the logic centralized rather than having each SectionPage detect its own position.
- **Gotcha**: The two-row footer uses two separate `<View fixed>` elements at different `bottom` offsets (32 and 20). React-PDF's `fixed` prop makes them repeat on every page of a wrapped `<Page>`.

**Anti-Persona (Compliance Officer) guard**: The anti-persona would want disclaimers on every page, hedging language in every section, and liability shields everywhere. We resist this — the confidentiality notice appears twice (cover + final page), the copyright line is institutional but minimal, and the advisory disclaimer is limited to the closing block. This is IP protection, not legal armor.
